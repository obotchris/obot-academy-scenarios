# Log In with the Bootstrap Token

Because authentication is enabled, Obot requires a one-time **bootstrap token** for the first login. You use it to sign in and configure an auth provider.

## Retrieve Your Bootstrap Token

The bootstrap token for your Obot instance is displayed on the launch page. Locate it in the environment panel and copy the token value — you will need it to log in.

## Open the Obot Admin UI

Open the Obot admin UI in your browser and go to the login page.

On the login page, click **Sign in with Bootstrap Token**, paste the token, and click **Login**.

You will land on the **Auth Providers** page, where you will configure GitHub in the next step.
