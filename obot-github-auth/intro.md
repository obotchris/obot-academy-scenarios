# Set Up GitHub Authentication in Obot

Obot is an open-source platform for implementing MCP (Model Context Protocol) technologies. When Obot runs with authentication enabled, users must sign in through an identity provider before they can access the platform.

In this scenario you will:

- Log in to the Obot instance with the bootstrap token
- Create a GitHub OAuth application
- Configure GitHub as an authentication provider in Obot
- Promote your GitHub account to **Owner**

## Prerequisites

- A GitHub account
- The ability to create an OAuth App under your GitHub developer settings

A running Obot instance with authentication enabled is provided for you in this environment — you do not need to install or start anything.
